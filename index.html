Algorithm Analysis — Hash Table
Player Lookup Alg 8 · Hash Table (Chaining)

Player IDs are hashed using djb2 into one of 16 buckets. Collisions are resolved via chaining. Each stat update is O(1) average. The grid below shows all 16 buckets live.

Enter a player ID and click Lookup.
Algorithm Analysis — Interval Tree
Who Was Batting? (Stabbing Query) Alg 9 · Interval Tree

Each batsman's spell is stored as an interval [entry over, dismissal over] in an augmented BST. To answer "who was batting at over X?", the tree prunes entire branches using the stored maxEnd value — no need to check every spell. Complexity: O(log n + k).

Enter an over and click Query.
Algorithm Analysis — AVL Tree Leaderboard
Top Scorers Alg 10 · AVL Self-Balancing BST

Every stat update inserts the player's new score into the AVL tree in O(log n). The tree auto-balances using rotations, keeping in-order traversal always sorted. No re-sort needed after each ball.
